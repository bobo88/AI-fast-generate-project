# 产品经理面试官话术指南

## 一、开场 (5分钟)

### 1.1 自我介绍

```
面试官:
"你好，我是XX，负责产品部门面试。今天我们大约会聊90分钟左右。
首先感谢你抽出时间来面试我们的产品经理岗位。

在整个面试过程中，我会根据你的经历问一些具体的问题，
有些问题可能需要你详细展开来说，这是在验证你过往的经历是否真实。
我们开始吧。"
```

### 1.2 暖场

```
"先简单了解一下你的背景情况。你目前是在职还是离职状态？"
"你目前主要负责什么类型的产品？"
"你为什么对这个岗位感兴趣？"
```

---

## 二、产品基础能力 (15分钟)

### 2.1 产品经理职责理解

**问题**: "你认为产品经理最重要的能力是什么？为什么？"

**参考话术**:
```
追问: "能举一个具体例子说明吗？"
追问: "那在你自己过往的工作中，你是怎么体现这个能力的？"
追问: "你觉得这个能力是天生的还是可以培养的？"
```

**评价要点**:
- 能否说出具体能力（如：需求分析、沟通协调、数据分析等）
- 能否解释为什么
- 能否举出真实案例

---

**问题**: "你做过的最复杂的产品是什么？请从背景、目标、用户、功能、结果完整讲一遍。"

**参考话术**:
```
追问: "这个产品有多少人参与？"
追问: "你在这个项目里负责什么？具体做了什么？"
追问: "这个项目做了多长时间？"
追问: "最后的结果怎么样？有数据吗？"
追问: "如果让你重新做一次，你会改进哪些地方？"
```

**评价要点**:
- 能否完整描述产品
- 能否说清自己的角色
- 能否给出具体数据

---

### 2.2 需求处理能力

**问题**: "当老板只告诉你'做一个远程控制机器人功能'，你会怎么开始？"

**参考话术**:
```
追问: "你会问哪些问题来明确需求？"
追问: "你会找哪些人沟通？"
追问: "第一步做什么？产出是什么？"
追问: "你怎么判断这个功能要不要做？"
```

**评价要点**:
- 是否会追问需求
- 是否会找用户/研发了解
- 是否有标准化流程

---

**问题**: "一个需求从提出到上线，你具体会怎么推进？"

**参考话术**:
```
追问: "需求评审是谁组织的？产出是什么？"
追问: "开发过程中你怎么跟进进度？"
追问: "测试阶段你做什么？"
追问: "上线后你怎么验证效果？"
```

---

## 三、硬件/机器人理解 (12分钟)

### 3.1 硬件经验了解

**问题**: "你是否做过和硬件、IoT、机器人相关的产品？"

**参考话术**:
```
(如果做过): "请详细介绍你负责的硬件产品。你负责哪部分？"
(如果没做过): "你对硬件产品有什么理解？你觉得和纯软件产品有什么区别？"
追问: "你觉得做硬件产品最大的挑战是什么？"
```

---

**问题**: "如果让你设计一个可以远程控制的小车机器人，你会考虑哪些模块？"

**参考话术**:
```
追问: "硬件部分需要哪些？"
追问: "软件部分需要哪些？"
追问: "云端需要什么？"
追问: "通讯层面怎么处理？"
追问: "如果要把成本降一半，你会怎么调整？"
```

**应有关键词**: 摄像头、云台、电机、电池、网络、GPS、避障、通讯协议、功耗

---

### 3.2 硬件场景问题

**问题**: "如果用户反馈'机器人有时候控制会延迟 2 秒'，你会怎么排查？"

**参考话术**:
```
追问: "你会找哪些人协助排查？"
追问: "可能的原因有哪些？从哪些层面分析？"
追问: "产品层面能做什么？"
追问: "这个问题能彻底解决吗？还是只能优化？"
```

---

**问题**: "机器人在家里断网了，产品上应该怎么设计？"

**参考话术**:
```
追问: "离线状态下用户能看到什么？"
追问: "机器人应该保持什么状态？"
追问: "恢复网络后怎么做？数据怎么同步？"
追问: "用户体验上怎么设计让用户知道断网了？"
```

---

**问题**: "如果机器人电量只剩 10%，系统应该如何处理？"

**参考话术**:
```
追问: "用户应该收到什么通知？"
追问: "机器人应该做什么动作？"
追问: "如果用户在远程操控过程中没电了怎么办？"
追问: "产品设计上要考虑哪些场景？"
```

---

### 3.3 高级场景设计

**问题**: "假设现在要做一个家庭巡逻机器人：可以远程查看监控、可以移动、可以自动巡逻、可以异常报警。请你拆出用户场景、核心功能、MVP、后续版本路线、最大风险。"

**参考话术**:
```
(边听边记录，观察候选人的结构)
追问: "你的目标用户是谁？"
追问: "他们有什么痛点？"
追问: "MVP怎么做？最小功能集是什么？"
追问: "后续版本怎么规划？"
追问: "最大的风险是什么？怎么规避？"
```

---

## 四、需求拆解与优先级 (10分钟)

### 4.1 优先级判断

**问题**: "现在有 20 个需求，但开发资源只够做 5 个，你怎么决定？"

**参考话术**:
```
追问: "你依据什么排序？"
追问: "有没有具体的方法论？比如Kano、四象限？"
追问: "排序的结果谁批准？"
追问: "被排后面的需求你怎么处理？"
追问: "如果有人不满意，你怎么说服？"
```

---

**问题**: "如果老板、客户、研发都在提不同需求，你怎么平衡？"

**参考话术**:
```
追问: "举一个具体的例子？"
追问: "他们的诉求分别是什么？"
追问: "有没有冲突的需求？你怎么协调？"
追问: "最后谁让步了？为什么？"
```

---

### 4.2 MVP定义

**问题**: "你通常如何定义MVP？"

**参考话术**:
```
追问: "MVP的最小功能集怎么确定？"
追问: "MVP怎么验证假设？"
追问: "验证后怎么迭代？"
追问: "你做过最成功的MVP是什么？"
```

---

### 4.3 真实验证

**问题**: "请举一个你真的砍掉过的需求。"

**参考话术**:
```
追问: "为什么砍？"
追问: "最后谁不满意？你怎么说服他们？"
追问: "如果被砍的需求后来证明是对的，你怎么办？"
追问: "你觉得砍需求的决策正确吗？"
```

---

## 五、结构化思维 (10分钟)

### 5.1 现场命题

**命题**: "我们准备做一个机器人远程看家产品，请你现场拆解。"

**参考话术**:
```
"这里有白板/纸笔，请你花10分钟时间，把你的想法写/画出来。"

(观察要点):
- 是否主动拆解: 用户→场景→问题→功能
- 是否有遗漏
- 逻辑是否清晰
- 是否有创新点

(追问):
追问: "你的用户是谁？"
追问: "他们的核心痛点是什么？"
追问: "MVP怎么做？"
追问: "最大的风险是什么？"
追问: "怎么验证产品是否成功？"
```

---

## 六、数据与结果导向 (8分钟)

### 6.1 指标理解

**问题**: "你做过的产品里，最重要的指标是什么？"

**参考话术**:
```
追问: "为什么是这个指标？"
追问: "这个指标怎么计算？"
追问: "目标值是多少？"
追问: "实际达成情况怎么样？"
```

---

**问题**: "你如何判断一个功能是否成功？"

**参考话术**:
```
追问: "你主要看哪些数据？"
追问: "数据从哪里来？"
追问: "成功的标准是什么？"
追问: "如果数据不理想，你怎么办？"
```

---

### 6.2 问题处理

**问题**: "如果一个新功能上线后，用户并没有使用，你会怎么办？"

**参考话术**:
```
追问: "你会分析什么？可能的原因有哪些？"
追问: "你会做什么决策？"
追问: "你是放弃还是优化？"
追问: "有没有具体的数据来支撑你的决策？"
```

---

### 6.3 失败案例

**问题**: "你之前项目中，有没有哪个指标没有达到目标？"

**参考话术**:
```
追问: "哪个指标？"
追问: "为什么？"
追问: "你做了什么？"
追问: "最后改善了吗？"
追问: "从这个案例中学到了什么？"
```

---

## 七、跨团队协作 (8分钟)

### 7.1 沟通难题

**问题**: "你合作过最难沟通的研发是什么情况？"

**参考话术**:
```
追问: "具体是什么问题？"
追问: "你当时是什么感受？"
追问: "你怎么解决的？"
追问: "最终结果如何？"
追问: "如果现在再遇到类似情况，你会怎么处理？"
```

---

**问题**: "研发说做不了，你怎么办？"

**参考话术**:
```
追问: "这是经常发生还是偶尔？"
追问: "你会怎么沟通？"
追问: "怎么判断是真的做不了还是不想做？"
追问: "有没有替代方案？"
追问: "最后怎么解决的？"
```

---

### 7.2 冲突处理

**问题**: "硬件延期、软件延期、测试延期时，你如何处理？"

**参考话术**:
```
追问: "举一个具体的例子？"
追问: "你当时做了什么？"
追问: "怎么协调各方？"
追问: "延期了怎么跟老板/客户交代？"
追问: "后来怎么解决的？"
```

---

**问题**: "如果两个部门意见完全冲突，你怎么推动？"

**参考话术**:
```
追问: "具体冲突是什么？"
追问: "你站在哪边？为什么？"
追问: "你怎么协调的？"
追问: "最后怎么解决的？"
追问: "双方满意吗？"
```

---

## 八、项目管理 (8分钟)

### 8.1 排期与进度

**问题**: "你通常如何排期？"

**参考话术**:
```
追问: "怎么评估工作量？"
追问: "怎么分配资源？"
追问: "排期工具用什么？"
追问: "遇到资源冲突怎么办？"
```

---

**问题**: "如何保证项目按时上线？"

**参考话术**:
```
追问: "里程碑怎么设置？"
追问: "风险怎么识别？"
追问: "进度怎么跟踪？"
追问: "遇到延期风险时你怎么做？"
```

---

### 8.2 问题处理

**问题**: "如果已经快上线了，但发现有一个严重问题，你会怎么办？"

**参考话术**:
```
追问: "问题的影响是什么？"
追问: "你会怎么决策？"
追问: "怎么处理与上线时间的冲突？"
追问: "你会上线还是延期？"
追问: "你怎么跟相关方沟通？"
```

---

### 8.3 大项目经验

**问题**: "你是否做过跨月、跨季度的大项目？"

**参考话术**:
```
追问: "有多少人在项目里？"
追问: "项目怎么分工？"
追问: "你最挑战的是什么？"
追问: "怎么协调各方进度的？"
追问: "最后项目成功了吗？"
```

---

## 九、真实水平验证 (12分钟)

### 9.1 深挖经历

**话术**:
```
"你说你做过XX功能，那请你:"
- "当时用户是谁？"
- "最开始的问题是什么？"
- "你亲自做了什么？"
- "研发多少人？"
- "做了多久？"
- "最后结果是什么？"

(如果答不出来): "这个项目你主要负责什么部分？"
(如果还是答不出): "那这个项目你参与度是怎样的？"
```

---

### 9.2 现场设计

**话术**:
```
"请你在10分钟内，现场设计一个'机器人远程看护系统'。
要求包括: 用户、场景、功能、页面、流程、风险、指标。"

(观察是否能在规定时间内完成)
(观察结构是否完整)
(观察是否有遗漏)

(追问): "为什么这样设计？"
(追问): "如果让你精简到只保留3个功能，你会保留什么？"
```

---

### 9.3 反向验证

**话术**:
```
"你刚才说这个功能很成功。"
- "那它失败的可能性是什么？"
- "如果重做一次，你会改什么？"
- "这个功能有什么缺点？"
- "你当时做了什么错误的决策？"

(真正做过的人，一定知道缺点和问题)
```

---

## 十、反向提问 (5分钟)

### 10.1 常规问题

```
"你有什么想了解我们公司的？"
"你对这个岗位有什么期望？"
"你的期望薪资是多少？"
"你最快什么时候可以入职？"
"你现在手里有其他Offer吗？"
```

### 10.2 结束语

```
"好的，今天的面试就到这里。你有什么问题想问我吗？"
"面试结果我们会在X个工作日内通知你。"
"感谢你今天来面试。"
```

---

## 十一、特殊情况处理

### 11.1 候选人过度紧张

**话术**:
```
"没关系，我们就是随便聊聊，放轻松。"
"这个问题答不上来也没关系，我们换个话题。"
"我理解你可能有点紧张，可以喝口水。"
```

### 11.2 候选人偏离主题

**话术**:
```
"这个话题很有意思，不过我们回到刚才的问题上。"
"你说的这个点我理解了，我们继续下一个问题。"
```

### 11.3 候选人过于能说

**话术**:
```
"好的，我理解了，那我们继续下一个问题。"
"你的经历很丰富，我们挑一个重点来说。"
```

### 11.4 候选人答不上来

**话术**:
```
"这个问题答不上来也没关系，我们聊聊其他话题。"
"你可以说说你的想法，不用担心对错。"
"那换个问题，你对XX方面有什么了解？"
```

---

## 十二、面试官注意事项

### 12.1 禁止事项

| 事项 | 说明 |
|------|------|
| 禁止诱导 | 不要给候选人提示答案 |
| 禁止偏见 | 不要因为候选人背景产生偏见 |
| 禁止打断 | 除非候选人严重偏离主题 |
| 禁止承诺 | 不要承诺录用或薪资 |
| 禁止负面 | 不要批评候选人的经历或能力 |

### 12.2 必做事项

| 事项 | 说明 |
|------|------|
| 全程记录 | 记录候选人回答的关键点 |
| 追问细节 | 遇到关键点要深挖 |
| 现场验证 | 必须让候选人现场设计/思考 |
| 反向验证 | 必须问失败案例 |
| 控制时间 | 每个环节控制好时间 |
